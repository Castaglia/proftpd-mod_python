ProFTPD module mod_python


The purpose of the mod_python module to is to provide a way to extend ProFTPD functionality using Python scripts.

Installation instructions are discussed here.

The most current version of mod_python can be found at: 

  https://github.com/Castaglia/proftpd-mod_python.git

PythonEngine

Syntax: PythonEngine on|off
Default: None
Context: server config, <VirtualHost>, <Global>
Module: mod_python
Compatibility: 1.3.6rc3 and later

The PythonEngine directive enables the Python support implemented by mod_python.

PythonLog

Syntax: PythonLog file
Default: None
Context: server config, <VirtualHost>, <Global>
Module: mod_python
Compatibility: 1.3.6rc3 and later

The PythonLog directive is used to specify a log file for mod_python's reporting on a per-server basis. The file parameter given must be the full path to the file to use for logging.

Installation

After unpacking latest proftpd-1.3.x source code, copy the mod_vroot.c file into: 
  proftpd-dir/contrib/
Then follow the normal steps for using third-party modules in proftpd: 
  $ ./configure --with-modules=mod_python
To build mod_python as a DSO module: 
  $ ./configure --enable-dso --with-shared=mod_python
Then follow the usual steps: 
  $ make
  $ make install
